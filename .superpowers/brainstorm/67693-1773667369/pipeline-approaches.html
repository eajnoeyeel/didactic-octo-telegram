파이프라인 아키텍처 — 3가지 접근법

전체 구조를 결정하는 핵심 선택입니다. 하나를 선택해주세요.

A

Sequential 2-Layer

서버 찾기 → 그 서버 안에서 Tool 찾기

쿼리
→
서버 인덱스 검색
Top-3 servers
→
1순위 서버 선택
→
Tool 인덱스 검색
(해당 서버 내)
→
Confidence 분기
Top-1 or Top-3
구현 단순  흐름 직관적  서버 선택 실수 시 Tool도 틀림
B

Parallel Dual-Index

서버/Tool 인덱스 동시 검색 → 통합 재랭킹

쿼리
→
서버 인덱스 검색
Tool 인덱스 검색
→
통합 Reranker
서버+Tool 점수 병합
→
Confidence 분기
빠름 (병렬)  서버 미등록 Tool도 발견 가능  점수 병합 로직 복잡
C

Taxonomy-gated Pipeline

의도 분류 먼저 → 해당 카테고리 내 검색 (JSPLIT 방식)

쿼리
→
인텐트 분류
web / data / code / ...
→
카테고리 필터
검색 공간 1/k
→
서버→Tool
Sequential
→
Confidence 분기
정밀도 높음  논문 근거 있음 (JSPLIT)  taxonomy 설계 필요  경계 모호 쿼리 취약

권고: 현재 규모(~1,000 tools)에서는 A (Sequential)이 가장 실용적입니다. C는 논문을 더 읽고 taxonomy 설계를 마친 뒤 Phase 2에서 추가할 수 있습니다. B는 Provider 분석(어떤 서버의 어떤 tool이 경쟁하는지)에 유리하지만 구현 복잡도가 높습니다.
