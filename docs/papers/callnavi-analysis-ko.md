# CallNavi

> 출처: 2025
> 한 줄 요약: LLM 도구 라우팅(올바른 도구 선택)에서 목적 명확성과 "언제 사용" 가이드가 가장 중요하며 파라미터 세부사항은 상대적으로 덜 중요함을 GPT-4o 기반 실험으로 밝힌 연구.

---

## 해결하려는 문제

LLM 에이전트가 여러 도구 중 올바른 도구를 선택(라우팅)하는 과정에서, 도구 설명의 어떤 요소가 가장 중요하게 작용하는지 분석. 라우팅 정확도와 파라미터 생성 정확도를 분리하여, 각 단계에서 설명의 어떤 부분이 기여하는지 규명.

## 핵심 아이디어

LLM의 도구 활용을 **라우팅(도구 선택)**과 **파라미터 생성(입력값 채우기)** 두 단계로 분리하여 분석. 라우팅 단계에서는 **목적 명확성(purpose clarity)**과 **"언제 사용하는가" 가이드라인**이 가장 중요하고, 파라미터 세부사항은 라우팅 정확도에 거의 기여하지 않음을 발견.

## 주요 결과

| 작업 | 정확도 (GPT-4o) |
|------|----------------|
| **라우팅** (올바른 도구명 선택) | **91.9%** |
| 파라미터 생성 (입력값 채우기) | 낮음 |

- 라우팅에는 **목적 명확성 + "언제 사용" 가이드**가 가장 중요
- 파라미터 세부사항은 라우팅 정확도에 **상대적으로 덜 중요**
- GPT-4o 기준 라우팅 정확도 91.9%로, 설명이 충분하면 도구 선택 자체는 높은 정확도 달성 가능

## 장점

- **단계 분리 분석**: 라우팅과 파라미터 생성을 분리하여 각 단계의 핵심 요인을 독립적으로 규명
- **가중치 설계 근거 제공**: DQS 차원별 가중치를 데이터 기반으로 설정할 수 있는 실증적 근거
- **실용적 통찰**: 도구 제공자가 제한된 시간에 어떤 설명 요소를 우선 개선해야 하는지 지침 제공

## 프로젝트 시사점

MCP Discovery Platform에서 DQS 차원별 가중치 설계의 핵심 근거를 제공하는 논문. 라우팅(도구 선택) 단계에서 purpose clarity가 가장 중요하다는 발견은, 우리 DQS에서 Purpose Clarity에 가장 높은 가중치(0.30)를 부여하고 Parameter Description에 가장 낮은 가중치(0.10)를 부여하는 직접적 근거가 됨.

## 적용 포인트

- **DQS 가중치 설계**: Purpose Clarity 0.30 (최고) / Parameter Description 0.10 (최저) 설정의 핵심 근거
- **Provider 피드백 우선순위**: 도구 설명 개선 시 "목적 + 사용 시점"을 최우선으로 안내하는 근거
- **라우팅 vs 파라미터 분리**: 향후 검색/추천 시스템에서 라우팅 단계와 실행 단계를 분리하여 최적화하는 아키텍처 설계 참고
- **E7 파일럿 기대치**: Purpose Clarity 차원이 가장 높은 Spearman 상관도를 보일 것이라는 가설의 근거
